# 株式会社MUSICO ホームページ 詳細仕様書

## 1. プロジェクト概要

### 1.1 基本情報
- **プロジェクト名**: musico-hp-renewal
- **開発目的**: 株式会社MUSICOのコーポレートサイトリニューアル
- **対象業務**: 企業向けBPO、カフェテリア運営、ホスピタリティサービス
- **開発期間**: 2024年〜
- **リリース環境**: Vercel Platform

### 1.2 技術スタック
- **フロントエンド**: Next.js 15.1.6, React 19.0.0
- **言語**: TypeScript 5.7.3
- **スタイリング**: Tailwind CSS 3.4.1, Framer Motion 11.18.1
- **バリデーション**: Zod 3.24.1
- **メール送信**: Resend 4.0.1, React Email 3.0.6
- **CMS**: microCMS
- **通知**: Slack Webhook
- **アニメーション**: Lottie React 2.4.1
- **コード品質**: Biome 1.9.4

## 4. ページ仕様

### 4.1 トップページ (`/`)
**URL**: `/ja/`, `/en/`
**目的**: 企業の第一印象、サービス概要の提示

**構成セクション**:
1. **Initial Animation**: Lottie アニメーション（初回のみ）
2. **First View**: メインビジュアル、キャッチコピー
3. **Services**: 3つの主要サービス紹介
4. **News**: 最新ニュース一覧（microCMS連携）
5. **Contact**: お問い合わせ誘導

**技術仕様**:
- Lottieアニメーション: `top-animation-new.json`
- 背景画像: `bg-fv-top.jpg`
- ニュース件数: 最大3件表示

### 4.2 会社概要ページ (`/about`)
**URL**: `/ja/about`, `/en/about`
**目的**: 企業情報の詳細提示

**表示項目**:
- 会社名: 株式会社MUSICO（ミュジコ）
- 代表取締役: 瀬本 頼一
- 資本金: 1,000万円
- 所在地: 東京都渋谷区宇田川町１番１２号 プラザ渋谷４階
- 主要取引銀行: 三菱ＵＦＪ銀行、みずほ銀行

### 4.3 お問い合わせページ (`/contact`)
**URL**: `/ja/contact`, `/en/contact`
**目的**: 見込み客からの問い合わせ受付

**フォーム項目**:
```typescript
{
  lastName: string,      // 姓（必須）
  firstName: string,     // 名（必須）
  lastNameKana: string,  // 姓カナ（必須）
  firstNameKana: string, // 名カナ（必須）
  organization: string,  // ご所属（必須）
  email: string,         // メールアドレス（必須、バリデーション）
  phoneNumber: string,   // 電話番号（必須、10-11桁）
  inquiryType: "サービスについて" | "採用について" | "その他", // 問い合わせ種別
  message: string        // お問い合わせ内容（必須、2000文字以内）
}
```

**バリデーション**:
- Zod スキーマによる厳密なバリデーション
- リアルタイムエラー表示
- 送信時の重複防止

**処理フロー**:
1. フォーム入力
2. クライアントサイドバリデーション
3. Server Action実行
4. 並列処理:
   - メール送信（Resend API）
   - Slack通知（Webhook）
5. 完了ページリダイレクト (`/contact/completed`)

### 4.4 ニュースページ (`/news`)
**URL**: `/ja/news`, `/en/news`, `/ja/news/[id]`, `/en/news/[id]`
**目的**: 企業からの情報発信

**一覧ページ機能**:

### 4.7 採用情報ページ (`/recruit`)
**URL**: `/ja/recruit`, `/en/recruit`
**目的**: 求職者への情報提供と応募受付

**募集職種**:
1. **バリスタ**: 
   - 業務: 外資系企業向けコーヒー・ティーサービス
   - スキル: コーヒー知識、接客経験
   
2. **シェフ**: 
   - 業務: 日替わりランチの調理・企画
   - スキル: 調理経験、メニュー開発
   
3. **パティシエ**: 
   - 業務: デザート・スイーツ全般
   - スキル: 製菓経験、創作能力
   
4. **ホスピタリティサービス**: 
   - 業務: 企業役員・海外ゲスト向け接遇
   - スキル: 接客経験、語学力
   
5. **パントリースタッフ**: 
   - 業務: バックオペレーション全般
   - スキル: 協調性、効率性
   
6. **プロジェクトマネージャー**: 
   - 業務: カフェ運営プロジェクト統括
   - スキル: 管理経験、リーダーシップ

**共通勤務条件**:
- 勤務地: 東京・港区エリア（外資系金融機関オフィス内）
- 勤務時間: 平日7:00〜17:00のシフト制
- 雇用形態: 正社員・契約社員・パート応相談
- 給与: 経験・能力により決定

### 4.8 採用応募ページ (`/recruit/apply/[jobId]`)
**URL**: `/ja/recruit/apply/[jobId]`, `/en/recruit/apply/[jobId]`
**目的**: 職種別応募受付

**応募フォーム項目**:
```typescript
{
  lastName: string,         // 姓（必須）
  firstName: string,        // 名（必須）
  lastNameKana: string,     // 姓カナ（必須）
  firstNameKana: string,    // 名カナ（必須）
  email: string,            // メールアドレス（必須）
  phoneNumber: string,      // 電話番号（必須）
  resume: File,            // 履歴書（PDF/DOC/DOCX、5MBまで）
  coverLetter: File,       // 職務経歴書（PDF/DOC/DOCX、5MBまで）
  motivation: string,      // 志望動機（任意、2000文字以内）
  jobId: string           // 応募職種ID
}
```

**ファイルアップロード仕様**:
- 対応形式: PDF, DOC, DOCX
- 最大サイズ: 5MB
- バリデーション: MIME Type + 拡張子チェック
- 一時保存: FormData経由でAPI送信

## 5. API仕様

### 5.1 メール送信API (`/api/email`)
**Method**: POST
**認証**: X-API-KEY ヘッダー
**Content-Type**: multipart/form-data

**リクエスト形式**:
```typescript
{
  type: "contact" | "recruit",
  formData: FormData,
  files?: File[]  // 採用応募時のみ
}
```

**レスポンス形式**:
```typescript
// 成功時
{
  success: true,
  messageId: string
}

// エラー時
{
  success: false,
  error: string
}
```

**処理フロー**:
1. 認証確認
2. フォームデータ解析
3. ファイル処理（採用時）
4. React Emailテンプレート生成
5. Resend API呼び出し
6. 結果返却

### 5.2 Slack通知仕様
**Webhook URL**: 環境変数 `SLACK_WEBHOOK_URL`
**通知タイミング**:
- お問い合わせ受信時
- 採用応募受信時
- システムエラー発生時

**通知内容**:
```typescript
// お問い合わせ通知
{
  text: "新しいお問い合わせを受信しました",
  blocks: [
    {
      type: "section",
      text: {
        type: "mrkdwn",
        text: `*お名前:* ${lastName} ${firstName}\n*会社名:* ${organization}\n*問い合わせ種別:* ${inquiryType}`
      }
    }
  ]
}
```

## 7. セキュリティ仕様

### 7.1 認証・認可
- **API認証**: X-API-KEY ヘッダー
- **環境分離**: 本番・開発環境の完全分離
- **秘密情報**: 環境変数での管理

### 7.2 入力値検証
- **Zod スキーマ**: 全フォーム項目の厳密な型チェック
- **ファイル検証**: MIME Type + 拡張子 + サイズ制限
- **XSS対策**: html-react-parser使用時の安全なHTML解析

### 7.3 通信セキュリティ
- **HTTPS**: 全通信の暗号化
- **CORS**: 必要最小限のオリジン許可
- **Rate Limiting**: API呼び出し制限（将来実装予定）

## 8. パフォーマンス仕様

### 8.1 画像最適化
