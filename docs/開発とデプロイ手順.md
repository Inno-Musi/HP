# MUSICO ホームページ リニューアル 開発・デプロイ手順

## 1. システム概要
- フレームワーク: Next.js 15 / React 19
- 言語: TypeScript
- スタイル: Tailwind CSS
- 外部サービス: microCMS（ニュース取得）、Resend（メール送信）、Slack Webhook（通知）、Google Tag Manager
- インフラ想定: Cloud Run（コンテナデプロイ）
- 永続データ: 外部サービス利用のためローカル DB は不要

## 2. 前提条件
### 2.1 必要ソフトウェア
- Node.js 18.18 以上（推奨 20.x）
- npm 10 以上
- Docker 24 以上（Cloud Run 用イメージビルドに使用）
- gcloud CLI（Cloud Run デプロイ時）

### 2.2 必要アカウント
- microCMS 管理者アカウント（`musico-hp` サービス）
- Resend アカウント（メール API キー取得）
- Slack 管理者権限（Incoming Webhook 作成）
- Google Tag Manager（GTM ID 管理）

## 3. 環境変数
`.env.local` を作成し、以下を設定してください（Git にはコミットしないでください）。

| 変数名 | 必須 | 用途 | ローカル例 |
| --- | --- | --- | --- |
| `NEXT_PUBLIC_VERCEL_ENV` | 必須 | 絶対 URL 生成時に本番/開発を判定 | `development` |
| `NEXT_PUBLIC_VERCEL_URL` | 必須 | アプリのホスト名（プロトコルなし） | `localhost:3000` |
| `GTM_ID` | 必須 | `<GoogleTagManager>` に渡す ID | `GTM-XXXXXXX` |
| `MICROCMS_API_KEY` | 必須 | ニュース記事取得用 API キー | `xxxxxxxxxxxxxxxxxxxx` |
| `RESEND_API_KEY` | 必須 | メール送信 API キー | `re_xxxxxxxxxxxxxxxxxx` |
| `SENDER_EMAIL` | 必須 | Resend 送信元 | `MUSICO Web <noreply@musico.co.jp>` |
| `X_API_KEY` | 必須 | `/api/email` エンドポイントの認証ヘッダー | ランダムな長い文字列 |
| `SLACK_WEBHOOK_URL` | 推奨 | 採用/問い合わせ通知用 Webhook | `https://hooks.slack.com/...` |

- Cloud Run 本番では `NEXT_PUBLIC_VERCEL_ENV=production` とし、`NEXT_PUBLIC_VERCEL_URL` はカスタムドメインまたは Cloud Run ドメイン（`https://` を除いた値）を指定してください。
- 秘密情報は Google Secret Manager に格納し、Cloud Run の変数として参照する運用を推奨します。

## 4. ローカル開発手順
1. 依存インストール
   ```bash
   npm ci
   ```
2. 環境変数を `.env.local` に設定
3. 開発サーバー起動
   ```bash
   npm run dev
   ```
4. `http://localhost:3000` で動作確認
5. Lint チェック（必要に応じて）
   ```bash
   npm run lint
   ```

### 4.1 データソースについて
- ニュース一覧・詳細は microCMS (`https://musico-hp.microcms.io/api/v1/news`) から取得します。ローカル DB は不要です。
- microCMS 上で公開済みコンテンツがない場合、サンプル記事を作成するか、スタブレスポンスを返すように microCMS 側でドラフトを用意してください。
- Slack Webhook を未設定の場合、通知処理は自動でスキップされます（`notifySlack` 内でエラーを返しつつ処理継続）。

## 5. Cloud Run デプロイ手順
### 5.1 Dockerfile 例
プロジェクトルートに `Dockerfile` を配置してください。
```dockerfile
# syntax=docker/dockerfile:1
FROM node:20-alpine AS deps
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci

FROM node:20-alpine AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN npm run build

FROM node:20-alpine AS runner
WORKDIR /app
ENV NODE_ENV=production
ENV NEXT_TELEMETRY_DISABLED=1
COPY --from=builder /app/public ./public
COPY --from=builder /app/.next ./.next
COPY package.json ./
COPY --from=deps /app/node_modules ./node_modules
EXPOSE 3000
CMD ["npm", "run", "start"]
```

### 5.2 コンテナビルド
```bash
gcloud builds submit --tag gcr.io/PROJECT_ID/musico-hp-renewal
```

### 5.3 Cloud Run へのデプロイ
```bash
gcloud run deploy musico-hp-renewal \
  --image gcr.io/PROJECT_ID/musico-hp-renewal \
  --platform managed \
  --region asia-northeast1 \
  --allow-unauthenticated \
  --set-env-vars NEXT_PUBLIC_VERCEL_ENV=production,NODE_ENV=production \
  --set-env-vars NEXT_PUBLIC_VERCEL_URL=example.com,GTM_ID=GTM-XXXXXXX \
  --set-env-vars MICROCMS_API_KEY=projects/PROJECT_ID/secrets/microcms_api_key/versions/latest \
  --set-env-vars RESEND_API_KEY=projects/PROJECT_ID/secrets/resend_api_key/versions/latest \
  --set-env-vars SENDER_EMAIL="MUSICO Web <noreply@musico.co.jp>" \
  --set-env-vars X_API_KEY=projects/PROJECT_ID/secrets/internal_x_api_key/versions/latest \
  --set-env-vars SLACK_WEBHOOK_URL=projects/PROJECT_ID/secrets/slack_webhook_url/versions/latest
```
- Secret Manager を利用する場合、`--set-env-vars` ではなく `--set-secrets` を使用しても構いません。
- Cloud Run が `PORT` 環境変数（デフォルト `8080`）を設定するため、追加設定は不要です。Next.js は自動でこのポートを使用します。

### 5.4 デプロイ後の確認
- サービス URL をブラウザで開き、言語切替・ニュース詳細・問い合わせフォーム送信を確認
- Resend のダッシュボードでメールログを確認
- Slack に通知が届くか確認
- Cloud Run ログビューアでエラー有無を確認（`gcloud logging read` またはコンソール）

## 6. 運用チェックリスト
- microCMS の API キーを更新した際は Cloud Run の環境変数も更新
- Resend 送信元ドメインの認証状態を定期確認
- Slack Webhook URL が再生成された場合は `.env.local` と Cloud Run の環境変数を同期
- 定期的に `npm run build` をローカルで実行し、依存関係の破損を早期検出
