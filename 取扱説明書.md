## 1. はじめに

### 1.1 本書の目的
この取扱説明書は、株式会社MUSICOのホームページを運用・管理するために必要な操作方法を説明したものです。

### 1.2 対象読者
- **管理者**: コンテンツ更新、システム設定、問い合わせ対応を行う方
- **運用担当者**: 日常的なコンテンツ管理を行う方
- **一般ユーザー**: サイト閲覧、お問い合わせ、応募を行う方

### 1.3 システム構成概要
- **フロントエンド**: Next.js ベースのWebサイト
- **CMS**: microCMS（ニュース管理）
- **メール**: Resend（自動送信）
- **通知**: Slack（管理者通知）
- **ホスティング**: Vercel

## 2. 管理者向け操作マニュアル

### 2.1 microCMS管理画面の操作

#### 2.1.1 ログイン方法
1. microCMS管理画面にアクセス
   - URL: `https://musico-hp.microcms.io/`
2. 管理者アカウントでログイン
3. 「musico-hp」プロジェクトを選択

#### 2.1.2 ニュース記事の作成
1. **新規記事作成**
   - サイドメニューの「コンテンツ」→「ニュース」をクリック
   - 「新規作成」ボタンをクリック

2. **記事内容入力**
   ```
   タイトル: 記事のタイトル（必須）
   本文: HTMLエディターで記事内容を入力
   公開日時: 記事の公開日時を設定
   ```

3. **記事の保存・公開**
   - 「下書き保存」: 非公開で保存
   - 「公開」: 即座にサイトに反映

#### 2.1.3 記事の編集・削除
1. **編集**: 記事一覧から編集したい記事をクリック
2. **削除**: 記事詳細画面の「削除」ボタンをクリック
3. **公開停止**: 公開状態を「下書き」に変更

#### 2.1.4 記事一覧の管理
- **検索**: タイトルや内容でのキーワード検索
- **並び替え**: 公開日、更新日、タイトル順での並び替え
- **フィルタ**: 公開・下書き状態での絞り込み

### 2.2 問い合わせ管理

#### 2.2.1 問い合わせ通知の受信
1. **Slackでの通知受信**
   - 新しい問い合わせが届くとSlackに自動通知
   - 通知内容: 氏名、会社名、問い合わせ種別

2. **メールでの詳細確認**
   - `info@musico.co.jp`に問い合わせ内容の詳細が届く
   - 添付ファイル（採用応募時）も含まれる

#### 2.2.2 問い合わせ対応フロー
1. **初回確認** (24時間以内)
   - 問い合わせ内容の確認
   - 担当部署への振り分け

2. **回答準備** (72時間以内)
   - 適切な回答内容の準備
   - 必要に応じて関係部署との調整

3. **回答送信**
   - 丁寧で具体的な回答を送信
   - 追加質問への対応準備

#### 2.2.3 採用応募の管理
1. **応募書類の確認**
   - 履歴書・職務経歴書のPDFファイル確認
   - 応募職種と志望動機の確認

2. **選考プロセス**
   - 応募者情報の人事システムへの登録
   - 書類選考結果の通知
   - 面接日程の調整

### 2.3 システム設定管理

#### 2.3.1 環境変数の管理
**Vercel Dashboard**での設定:
```
MICROCMS_API_KEY: microCMSのAPIキー
RESEND_API_KEY: ResendのAPIキー
SLACK_WEBHOOK_URL: SlackのWebhook URL
X_API_KEY: 内部API認証キー
GTM_ID: Google Tag ManagerのID
SENDER_EMAIL: 送信元メールアドレス
```

#### 2.3.2 ドメイン・SSL管理
1. **カスタムドメイン設定**
   - Vercel Dashboard → Settings → Domains
   - 会社ドメインの設定と検証

2. **SSL証明書**
   - 自動更新設定の確認
   - 証明書の有効期限監視

#### 2.3.3 アクセス解析の確認
1. **Google Analytics**
   - Google Tag Managerを通じたデータ収集
   - 月次レポートの作成

2. **Vercel Analytics**
   - パフォーマンス指標の監視
   - エラー発生状況の確認

## 3. 一般ユーザー向け操作ガイド

### 3.1 サイトの基本操作

#### 3.1.1 言語切替
1. **日本語⇔英語の切り替え**
   - ヘッダー右上の言語切替ボタンをクリック
   - 「日本語」または「English」を選択

2. **自動言語判定**
   - 初回アクセス時は日本語で表示
   - ブラウザ設定に応じた言語表示（将来実装予定）

#### 3.1.2 ナビゲーション
1. **ヘッダーメニュー**
   ```
   会社概要 (About)
   サービス (Services)  
   企業理念 (Philosophy)
   ニュース (News)
   採用情報 (Recruit)
   お問い合わせ (Contact)
   ```

2. **フッターリンク**
   - プライバシーポリシー
   - セキュリティポリシー
   - 会社情報

#### 3.1.3 レスポンシブ対応
- **PC**: フルレイアウトで表示
- **タブレット**: 最適化されたレイアウト  
- **スマートフォン**: ハンバーガーメニュー

### 3.2 お問い合わせの方法

#### 3.2.1 お問い合わせフォームの入力
1. **基本情報の入力**
   ```
   ✓ 姓・名 (必須)
   ✓ 姓カナ・名カナ (必須)
   ✓ ご所属 (必須)
   ✓ メールアドレス (必須)
   ✓ 電話番号 (必須、ハイフンなし10-11桁)
   ```

2. **問い合わせ内容**
   ```
   問い合わせ種別:
   - サービスについて
   - 採用について  
   - その他
   
   お問い合わせ内容: 2000文字以内で詳細を記載
   ```

#### 3.2.2 入力時の注意点
- **必須項目**: 赤い*マークが付いた項目は必須
- **メール形式**: 正しいメールアドレス形式で入力
- **電話番号**: ハイフンなしの半角数字のみ
- **文字数制限**: お問い合わせ内容は2000文字以内

#### 3.2.3 送信後の流れ
1. **送信完了**: 完了ページが表示される
2. **自動返信**: 入力したメールアドレスに確認メールが送信
3. **管理者対応**: 72時間以内に担当者から回答

### 3.3 採用応募の方法

#### 3.3.1 募集職種の確認
1. **採用情報ページ**で職種一覧を確認
2. **詳細情報**をクリックして業務内容・応募条件を確認
3. **応募する**ボタンをクリック

#### 3.3.2 応募フォームの入力
1. **基本情報**
   ```
   ✓ 姓・名 (必須)
   ✓ 姓カナ・名カナ (必須)  
   ✓ メールアドレス (必須)
   ✓ 電話番号 (必須)
   ```

2. **応募書類**
   ```
   ✓ 履歴書 (必須)
     - 形式: PDF, DOC, DOCX
     - サイズ: 5MB以内
   
   ✓ 職務経歴書 (必須)
     - 形式: PDF, DOC, DOCX  
     - サイズ: 5MB以内
   ```

3. **志望動機** (任意)
   - 2000文字以内で記載

#### 3.3.3 応募後の選考フロー
1. **応募受付**: 自動確認メールが送信
2. **書類選考**: 1週間程度で結果連絡
3. **面接**: 合格者のみ面接日程を調整
4. **最終結果**: 面接後1週間程度で連絡

## 4. コンテンツ更新・追加方法

### 4.1 ニュース記事の管理

#### 4.1.1 記事作成のベストプラクティス
1. **タイトル作成**
   - 32文字以内で内容を正確に表現
   - SEOを意識したキーワード使用
   - 日付は本文に含める（タイトルに不要）

2. **本文作成**
   - 見出し（H2, H3）を使用した構造化
   - 適切な段落分け
   - 画像使用時はalt属性を設定

3. **公開タイミング**
   - 重要なお知らせ: 即座に公開
   - 一般的なニュース: 平日10:00-16:00に公開
   - イベント情報: 2週間前までに公開

#### 4.1.2 画像の追加方法
1. **microCMS メディアライブラリ**
   - 「メディア」タブから画像をアップロード
   - 推奨サイズ: 横1200px以上
   - 形式: JPEG, PNG, WebP

2. **記事内での使用**
   - エディターの「画像」ボタンをクリック
   - メディアライブラリから選択
   - alt属性（代替テキスト）を必ず設定

#### 4.1.3 記事の削除・アーカイブ
1. **古い記事の整理**
   - 3か月以上前の一般ニュースは削除検討
   - 重要な発表は削除せず「アーカイブ」タグを付与

2. **削除前の確認事項**
   - 外部リンクされていないか確認
   - 重要な情報が含まれていないか確認

### 4.2 会社情報の更新

#### 4.2.1 会社概要の変更
**ファイル**: `src/features/about/`
```typescript
// 会社情報の更新箇所
const companyInfo = {
  name: "株式会社MUSICO",
  representative: "瀬本 頼一",
  capital: "1,000万円",
  address: "東京都渋谷区宇田川町１番１２号 プラザ渋谷４階",
  // 変更時はこちらを更新
}
```

#### 4.2.2 サービス内容の更新
**ファイル**: `src/features/services/`
- サービス説明文の変更
- サービス画像の差し替え
- 新サービスの追加

#### 4.2.3 採用情報の更新
**ファイル**: `src/features/recruit/`
1. **新職種追加**
   - job定義に新職種を追加
   - 日英両方の情報を必須で追加

2. **募集条件変更**
   - 給与、勤務時間等の条件変更
   - 応募資格の変更

### 4.3 多言語コンテンツの管理

#### 4.3.1 翻訳作業のフロー
1. **日本語版作成**: まず日本語でコンテンツを作成
2. **英語翻訳**: 専門的な用語は統一性を保つ
3. **ネイティブチェック**: 可能であれば英語ネイティブによるチェック
4. **両言語同時更新**: 日英同時に更新して整合性を保つ

#### 4.3.2 翻訳時の注意点
- **専門用語**: 一貫した翻訳を使用
- **文化的配慮**: 日本と海外の文化差を考慮
- **文字数**: 英語は日本語より長くなる傾向

## 5. トラブルシューティング

### 5.1 よくある問題と解決方法

#### 5.1.1 問い合わせフォームの問題
**問題**: フォーム送信エラー
```
解決手順:
1. 入力内容の確認
   - 必須項目がすべて入力済みか
   - メール形式が正しいか
   - 電話番号が半角数字のみか

2. ブラウザの確認
   - キャッシュをクリア
   - JavaScript が有効か
   - 別のブラウザで試行

3. 継続する場合
   - エラーメッセージのスクリーンショット
   - ブラウザ・OSの情報
   - 管理者に報告
```

#### 5.1.2 ファイルアップロードの問題
**問題**: 履歴書・職務経歴書がアップロードできない
```
確認項目:
✓ ファイル形式: PDF, DOC, DOCX のみ
✓ ファイルサイズ: 5MB以内
✓ ファイル名: 日本語・特殊文字を避ける
✓ ネットワーク: 安定した接続環境

解決方法:
1. ファイル形式をPDFに変換
2. ファイルサイズを圧縮
3. ファイル名を英数字に変更
```

#### 5.1.3 ページ表示の問題
**問題**: ページが正しく表示されない
```
確認手順:
1. ブラウザのリロード (Ctrl+F5 / Cmd+Shift+R)
2. キャッシュのクリア
3. 別のブラウザで確認
4. 同様の問題が継続する場合は管理者に報告
```

### 5.2 管理者向けトラブルシューティング

#### 5.2.1 microCMS接続エラー
```
確認項目:
1. APIキーの有効性確認
2. microCMSサービス状況確認
3. ネットワーク接続確認
4. Vercelログの確認

解決手順:
1. APIキーの再発行
2. 環境変数の更新
3. デプロイメント実行
```

#### 5.2.2 メール送信エラー
```
確認項目:
1. Resend APIキーの確認
2. 送信元メールアドレスの認証状況
3. 送信制限の確認

解決手順:
1. Resendダッシュボードでの確認
2. APIキーの更新
3. 送信元ドメインの再認証
```

#### 5.2.3 Slack通知が届かない
```
確認項目:
1. Webhook URLの有効性
2. Slackチャンネルの権限
3. 通知設定の確認

解決手順:
1. 新しいWebhook URLの生成
2. 環境変数の更新
3. テスト通知の送信
```
